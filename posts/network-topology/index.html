Network Topology

네트워크 구성 설명하기

February 16, 2025

  • network

네트워크 구성 방식에는 여러 가지가 있으며, 각 방식은 고유한 장단점을 가지고 있습니다. 주요 네트워크 구성 방식과 그들의 장단점을 그래프와 함께 설명드릴게요.

1. 스타형 (Star Topology)

      +----+
      |Hub |
      +--+-+
         | 
  +------+------+
  |      |      |
+---+  +---+  +---+
|PC1|  |PC2|  |PC3|
+---+  +---+  +---+

장점:

  • 중앙 허브 또는 스위치를 통해 모든 노드가 연결되어 있어 관리가 용이합니다.
  • 중앙 장치에 문제가 없는 한 각 노드의 장애가 전체 네트워크에 영향을 미치지 않습니다.
  • 새로운 장치를 추가하거나 제거하는 것이 간단합니다.

단점:

  • 중앙 장치가 고장 나면 전체 네트워크가 중단됩니다.
  • 설치 비용이 많이 들 수 있습니다.
  • 중앙 장치에 트래픽이 집중되어 성능 저하가 발생할 수 있습니다.

2. 링형 (Ring Topology)

+---+    +---+
|PC1|----|PC2|
+---+    +---+
  |        |
  |        |
+---+    +---+
|PC4|----|PC3|
+---+    +---+

장점:

  • 데이터 전송이 고정된 경로를 따라 순환하므로 충돌이 적습니다.
  • 중간 노드에서 데이터를 중계하여 신호가 약해지지 않습니다.

단점:

  • 한 노드가 고장 나면 전체 네트워크가 중단될 수 있습니다.
  • 데이터 전송 지연이 발생할 수 있습니다.
  • 네트워크 확장이 어려울 수 있습니다.

3. 버스형 (Bus Topology)

+---+    +---+    +---+
|PC1|----|PC2|----|PC3|
+---+    +---+    +---+
         |
       +---+
       |PC4|
       +---+

장점:

  • 설치와 확장이 비교적 간단하고 저렴합니다.
  • 케이블 사용량이 적어 비용 효율적입니다.

단점:

  • 중앙 버스가 고장 나면 전체 네트워크가 중단됩니다.
  • 트래픽이 증가하면 성능이 저하될 수 있습니다.
  • 장애 발생 시 문제를 찾기가 어려울 수 있습니다.

4. 망형 (Mesh Topology)

   +---+    +---+
  /|PC1|----|PC2|\
 / +---+    +---+ \
/    |        |    \
+----|--------|----+
|   +---+    +---+  |
|   |PC4|----|PC3|  |
|   +---+    +---+  |
 \    |        |    /
  \   +---+    +---+
   \--|PC5|----|PC6|
      +---+    +---+

장점:

  • 모든 노드가 서로 연결되어 있어 높은 신뢰성을 제공합니다.
  • 한 경로가 고장 나더라도 다른 경로로 데이터 전송이 가능합니다.
  • 확장성이 뛰어납니다.

단점:

  • 설치와 유지보수가 매우 복잡하고 비용이 많이 듭니다.
  • 케이블과 포트 수가 많아져 하드웨어 비용이 증가할 수 있습니다.

5. 트리형 (Tree Topology)

        +---+
        |Hub|
        +--+-+
           |
  +--------+--------+
  |                 |
+---+             +---+
|S1 |             |S2 |
+---+             +---+
  |                 |
+---+             +---+
|PC1|             |PC2|
+---+             +---+
  |
+---+
|PC3|
+---+

장점:

  • 계층 구조로 확장성과 유연성이 좋습니다.
  • 부분적인 네트워크 장애가 전체 네트워크에 미치는 영향을 최소화할 수 있습니다.

단점:

  • 루트 노드 또는 주요 중계 노드가 고장 나면 네트워크 전체에 영향을 줄 수 있습니다.
  • 관리와 유지보수가 복잡할 수 있습니다.
      • 1. 스타형 (Star Topology)
      • 2. 링형 (Ring Topology)
      • 3. 버스형 (Bus Topology)
      • 4. 망형 (Mesh Topology)
      • 5. 트리형 (Tree Topology)
